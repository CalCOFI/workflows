# CalCOFI.io Landing Page Redesign

## Context

The calcofi.io landing page (repo `CalCOFI/CalCOFI.github.io`) is an R Markdown + `bs4cards` page driven by a Google Sheet, manually rendered to a committed 17 MB `index.html` ŌĆö 16 products in one alphabetical grid with no audience guidance, dark Bootswatch theme inconsistent with the newer family of sites. Meanwhile the ecosystem has grown (int-app, ctd-viz, datacheck, query, schema, workflows site, calcofi4db, 2026 student capstones) and schema/, query/, and workflows/_output now share a clean YAML-driven Jekyll style.

**Goal:** rebuild the landing page as a Jekyll site in that family style, organized as audience pathways (featured ŌåÆ Explore ŌåÆ Access Data ŌåÆ Build ŌåÆ Student Contributions), products defined in version-controlled `_data/products.yml` (retiring the Google Sheet), with the two new 2026 student projects hosted via GitHub Pages and added to the page.

User decisions (confirmed): Jekyll YAML-driven ┬Ę audience pathways + featured hero ┬Ę GitHub Pages for both student repos ┬Ę legacy apps demoted-but-visible with badges.

Verified facts grounding this plan:
- `gh api repos/CalCOFI/CalCOFI.github.io/pages` ŌåÆ `build_type: legacy` (branch main, `/`), `cname: calcofi.io`; schema/query/workflows use `build_type: workflow` (Actions). Caddy serves only subdomains, so GitHub Pages safely owns the apex.
- **`uptime/.upptimerc.yml` is the authoritative inventory of live apps/services.** Canonical Shiny URL pattern: `https://app.calcofi.io/<app>/`. Production apps: oceano, **ctd** (= ctd-viz, live at `app.calcofi.io/ctd/`), datacheck, dashboard, **copernicus**, int, marmam, pollutants, SaferSeafood, capstone. Infrastructure monitored: shiny server, tile server, **file server (`file.calcofi.io`)**, home.
- Canonical URLs: `calcofi.io/workflows/`, `status.calcofi.io` (alternates 301 to these).
- UCSB portal: fully static, publish dir `public/`, all paths relative ŌåÆ subpath-safe, no code changes.
- UCLA map: publish dir `web/`, single `index.html`; `web/index.html:208` has `const DATA_BASE = ''` ŌĆö must point at public GCS bucket. All fetches resolve off `DATA_BASE` (lines 381, 511).

### Page design
- **Hero** ("from ship to screen" pipeline narrative): CalCOFI cruises ŌåÆ reproducible workflows ŌåÆ versioned integrated database ŌåÆ apps ┬Ę API ┬Ę R package; audiences named (scientists, managers, students, public). Small mono pipeline line: `ship ŌåÆ workflows ŌåÆ database ŌåÆ apps ┬Ę API ┬Ę R`.
- **Featured row**: larger cards (minmax ~420px) for int-app + ctd-viz with tech chips.
- **Section-nav chips** (in-page anchors `#explore #data #build #students`), then sections each with emoji, h2, blurb, responsive `auto-fill minmax(300px,1fr)` card grid.
- **Card**: 16:10 `object-fit: cover` screenshot, title link, status badge (superseded badge links to `superseded_by` product via `where: "key"` lookup), description, tech chips, student credits line, `open ŌåŚ / source ŌåŚ` links. `loading="lazy"` images; `.demoted { opacity: .75 }`.
- **Layout**: keep family theme toggle (dark default + light), sticky header; nav = in-page anchors. GA snippet in head gated on `jekyll.environment == "production"`, tag `G-0HVK8TDMCF`.
- **Footer**: Program ┬Ę Erin Satterthwaite (esatterthwaite@ucsd.edu) ┬Ę Technical ┬Ę Ben Best (ben@ecoquants.com) ┬Ę CalCOFI.org ┬Ę github.com/CalCOFI ┬Ę status.

Jekyll version: pinned Jekyll 4.3 via Gemfile + Actions build (same as schema/query) ŌĆö free to use Jekyll 4 features; no legacy-Pages compatibility constraint (user accepts a brief cutover gap, and flipping the Pages source before merge avoids any broken-build window anyway).
