1 CalCOFI.io Landing Page Redesign

1.1 Context

The calcofi.io landing page (repo CalCOFI/CalCOFI.github.io) is an R Markdown + bs4cards page driven by a Google Sheet, manually rendered to a committed 17 MB index.html — 16 products in one alphabetical grid with no audience guidance, dark Bootswatch theme inconsistent with the newer family of sites. Meanwhile the ecosystem has grown (int-app, ctd-viz, datacheck, query, schema, workflows site, calcofi4db, 2026 student capstones) and schema/, query/, and workflows/_output now share a clean YAML-driven Jekyll style.

Goal: rebuild the landing page as a Jekyll site in that family style, organized as audience pathways (featured → Explore → Access Data → Build → Student Contributions), products defined in version-controlled _data/products.yml (retiring the Google Sheet), with the two new 2026 student projects hosted via GitHub Pages and added to the page.

User decisions (confirmed): Jekyll YAML-driven · audience pathways + featured hero · GitHub Pages for both student repos · legacy apps demoted-but-visible with badges.

Verified facts grounding this plan: - gh api repos/CalCOFI/CalCOFI.github.io/pages → build_type: legacy (branch main, /), cname: calcofi.io; schema/query/workflows use build_type: workflow (Actions). Caddy serves only subdomains, so GitHub Pages safely owns the apex. - uptime/.upptimerc.yml is the authoritative inventory of live apps/services. Canonical Shiny URL pattern: https://app.calcofi.io/<app>/. Production apps: oceano, ctd (= ctd-viz, live at app.calcofi.io/ctd/), datacheck, dashboard, copernicus, int, marmam, pollutants, SaferSeafood, capstone. Infrastructure monitored: shiny server, tile server, file server (file.calcofi.io), home. - Canonical URLs: calcofi.io/workflows/, status.calcofi.io (alternates 301 to these). - UCSB portal: fully static, publish dir public/, all paths relative → subpath-safe, no code changes. - UCLA map: publish dir web/, single index.html; web/index.html:208 has const DATA_BASE = '' — must point at public GCS bucket. All fetches resolve off DATA_BASE (lines 381, 511).

# CalCOFI.io Landing Page Redesign

## Context

The calcofi.io landing page (repo `CalCOFI/CalCOFI.github.io`) is an R Markdown + `bs4cards` page driven by a Google Sheet, manually rendered to a committed 17 MB `index.html` — 16 products in one alphabetical grid with no audience guidance, dark Bootswatch theme inconsistent with the newer family of sites. Meanwhile the ecosystem has grown (int-app, ctd-viz, datacheck, query, schema, workflows site, calcofi4db, 2026 student capstones) and schema/, query/, and workflows/_output now share a clean YAML-driven Jekyll style.

**Goal:** rebuild the landing page as a Jekyll site in that family style, organized as audience pathways (featured → Explore → Access Data → Build → Student Contributions), products defined in version-controlled `_data/products.yml` (retiring the Google Sheet), with the two new 2026 student projects hosted via GitHub Pages and added to the page.

User decisions (confirmed): Jekyll YAML-driven · audience pathways + featured hero · GitHub Pages for both student repos · legacy apps demoted-but-visible with badges.

Verified facts grounding this plan:
- `gh api repos/CalCOFI/CalCOFI.github.io/pages` → `build_type: legacy` (branch main, `/`), `cname: calcofi.io`; schema/query/workflows use `build_type: workflow` (Actions). Caddy serves only subdomains, so GitHub Pages safely owns the apex.

Production apps: oceano, **ctd** (= ctd-viz, live at `app.calcofi.io/ctd/`), datacheck, dashboard, **copernicus**, int, marmam, pollutants, SaferSeafood, capstone. Infrastructure monitored: shiny server, tile server, **file server (`file.calcofi.io`)**, home.
